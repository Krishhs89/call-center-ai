================================================================================
AGENT DETAILS
================================================================================

CORE AGENTS (V1):
-----------------
intake_agent.py
  - Validates and processes input calls (text, JSON, audio path)
  - Extracts metadata: call_id, category, duration, timestamp
  - Creates CallInput with Pydantic

transcription_agent.py
  - OpenAI Whisper for audio files (WAV/MP3)
  - Normalises speaker labels -> "Agent" / "Customer"
  - Fallback to raw text if Whisper unavailable

summarization_agent.py
  - Configurable LLM (Claude/GPT-4/Gemini)
  - Structured output: summary, key_points, action_items, resolution_status
  - Temperature: 0.3

quality_score_agent.py
  - 6-dimension scoring: empathy, professionalism, resolution,
    compliance, communication, efficiency
  - Overall score 0-100; strengths + improvement suggestions
  - Temperature: 0.2

routing_agent.py
  - State transition logging (LangSmith-compatible)
  - Error recovery routing


V2 AGENTS:
----------
pii_redaction_agent.py
  - Zero LLM cost -- pure regex
  - Masks: phone, email, SSN, credit card, dates of birth, ZIP codes
  - Returns redacted transcript + PII audit log

rag_retrieval_agent.py
  - ChromaDB vector search (OpenAI text-embedding-3-small)
  - Returns top-3 most similar past calls with similarity scores
  - Embeds PII-redacted transcripts only

sentiment_agent.py
  - Turn-by-turn sentiment analysis
  - Overall sentiment trend, escalation_risk score
  - Agent tone classification
  - Temperature: 0.2


V3 AGENTS:
----------
compliance_checker_agent.py
  - 6 compliance frameworks: HIPAA, GDPR, PCI-DSS, TCPA, Financial, General
  - Output: violations (severity + evidence + remediation), compliance_score 0-100
  - requires_immediate_review flag
  - Mock: pattern-based (CVV, recording consent, identity verification keywords)

escalation_prediction_agent.py
  - Cross-agent signals: sentiment + compliance
  - risk_score 0-100, risk_level (low/medium/high/critical)
  - trigger_moments list, recommended_intervention
  - Temperature: 0.3

call_coaching_agent.py
  - Tied to weakest QA dimension_scores
  - Output: agent_strengths, coaching_tips (each with example_script),
    next_call_focus, estimated_improvement, overall_coaching_priority
  - Temperature: 0.4

knowledge_base_agent.py
  - Dual mode: retrieve_context() for prompt injection + process() for SOP audit
  - 7 built-in KB articles: refund, verification, escalation, billing,
    tech support, recording consent, shipping
  - Output: relevant_articles, sop_compliance_score, missed_knowledge_opportunities
  - Temperature: 0.1

customer_profile_agent.py
  - ZERO LLM COST -- pure JSONL history aggregation
  - Risk tiers: vip / at_risk / churning / regular
  - Output: total_calls, avg_qa_score, escalation_count, sentiment_trend,
    risk_tier, top_issues, is_first_call

auto_tagging_agent.py
  - 5-taxonomy multi-label classification
  - Output: primary_category, sub_category, intent_tags, sentiment_tags,
    routing_tags, product_tags, confidence_score
  - Temperature: 0.1

anomaly_detection_agent.py
  - ZERO LLM COST -- statistical z-score + rule-based thresholds
  - Flags: low_qa_score, critical_compliance, multiple_violations,
    high_escalation_risk, extreme_sentiment, excessive_pii
  - Output: anomaly_score 0-100, anomaly_level, flags, requires_review

feedback_loop_agent.py
  - ZERO LLM COST -- compares current vs last 3 calls from JSONL
  - Output: improvement_status (significantly_improved/improved/stable/declined),
    score_delta, coaching_adoption_rate, improved_dimensions, regressed_dimensions


================================================================================
WORKFLOW -- langgraph_flow.py
================================================================================

Pipeline (17 nodes + error branch):

  intake -> customer_profile -> transcription -> pii_redaction -> rag_retrieval
         -> kb_retrieval -> sentiment -> compliance_check -> escalation_prediction
         -> summarization -> auto_tagging -> quality_score -> call_coaching
         -> anomaly_detection -> end

  Post-workflow: FeedbackLoopAgent runs after CallResult assembled

State: WorkflowState (TypedDict) carries 20+ fields through the graph

_v2_extras dict carries 13 keys to UI:
  sentiment_analysis, pii_audit, rag_context, customer_profile,
  kb_context, kb_analysis, compliance_check, escalation_prediction,
  tags, coaching, anomaly, feedback_loop, rag_used
